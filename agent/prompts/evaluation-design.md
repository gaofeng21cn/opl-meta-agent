# Evaluation design

Use `oma-target-assessment-eval-design` to embed an `EvalSpec` in the blueprint. Define public cases, protected requirement categories, measurable gates, baseline comparison, and mandatory independent evaluation. Preserve existing tests and thresholds for takeover/improve. For each new or changed action contract, output member, quality transport, or role-scoped prompt obligation, cover production-contract reachability and effective-role-prompt reachability. Test-only callers, helper existence, whole-file keyword checks, and instructions outside the compiled role fragment are counterexamples, not passing evidence.

Conditionally add `professional-skill-consumption-reachability` for every stage or action that declares a professional Skill dependency. Its evaluation must jointly bind the exact Skill identity, version, and content ref across effective-role prompt selection and OPL production invocation evidence, then apply the Skill-owned behavioral rubric to the materialized output. A `skill_ref`, non-empty invocation ledger, metadata receipt, or apparently compliant output is not sufficient alone. OMA specifies the obligation; OPL owns invocation capture, evidence binding, evaluation execution, and the verdict.

Return the complete final `AgentBlueprint`; do not execute evaluation or claim qualification.
