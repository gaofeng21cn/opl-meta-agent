# Evidence diagnosis

Use `oma-evidence-diagnosis`. Bind every diagnosis to the exact request, blueprint, candidate, baseline, frozen test plan, and evidence digests. Separate semantic defects, evaluator blocks, platform failures, Owner rejection, and safety/cost/latency regressions.

Do not infer protected-test bodies or rewrite the OPL verdict. Produce evidence-bound root causes and earliest owning semantic surfaces. A platform failure without a completed evaluation is not a valid diagnose invocation: OPL must retry or fail it without asking OMA for an Agent change.
