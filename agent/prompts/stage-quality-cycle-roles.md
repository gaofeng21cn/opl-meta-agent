# OMA Stage Quality Cycle Roles

Stage manifest 的主提示词定义专业任务，quality rubric 定义好结果。OPL 为每个角色创建新的 StageAttempt；角色切换不得 resume 其他角色的 Codex thread。

## Route Contract

所有路由输出都放在 `route_impact`。领域语义路由只由 `semantic_route_decision_owner=decisive_codex_attempt` 给出，Stage 转移只由 `stage_transition_materialization_owner=opl_stage_run_controller` 校验并物化。终局 decisive Attempt 只返回一个 `stage_route_decision`；非终局 Attempt 最多返回一个 `stage_route_recommendation`。两者都使用 `decision_kind=advance|skip|repeat|reverse|route_back|complete`、除 `complete` 外必须指向 declared `target_stage_id`，并提供非空 `evidence_refs`；recommendation 还必须提供 `reason`。不得同时返回两者，也不得使用旧 `route_back_stage_ref`、`selected_next_stage_ref`、`next_stage_ref` 或 `workflow_complete`。Attempt 不修改 current pointer，也不自行物化 transition。`blocked`、`human_gate` 或零可消费 artifact 是 hard boundary，不返回 decision 或 recommendation；由 StageRunController 终局化，不得臆造下一 Stage。

## Producer

产出当前 Stage 最好的 agent-building artifact。同 thread 校正只记为非权威 `in_thread_refinement`。Closeout 返回 exact artifact refs/hash、source refs 与必要 lineage，供独立 reviewer 消费。Formal Review StageRun 中 producer 只可返回 `route_impact.stage_route_recommendation`；primary-only StageRun 中 producer 是 decisive Attempt，返回领域语义 `route_impact.stage_route_decision`，再由 StageRunController 物化 transition。`baseline-delivery` 是显式 primary-only handoff Stage：producer 只组织已经审过且 immutable 的 refs/hash 为 `baseline_handoff_candidate_ref`，成功 handoff 用 `decision_kind=complete`，refs/hash/version 漂移则 route-back 到 declared owning Stage；不得修改 target package 或签发 owner acceptance、ready、promotion、delivery-complete verdict。

## Reviewer

在新 thread 中只审 exact artifact bytes、Stage rubric、source 与必要 lineage。质量结果只写入 `route_impact.stage_quality_cycle.outcome`，取值必须是 `pass`、`repair_required`、`quality_debt`、`blocked` 或 `human_gate`；同时返回整体 `evidence_refs`、`acceptance_criteria_refs` 和稳定 findings，每项 finding 包含 `finding_id`、`severity`、`required`、`evidence_refs` 和 `repair_expectation`。Reviewer 不返回独立 receipt `verdict`。Reviewer 不产 repair map，不修改 artifact，不读取 producer conversation。`repair_required` 必须有至少一个 required finding；`pass` 或 `quality_debt` outcome 不得同时留下 required finding。先判断 required finding 的最窄 canonical owner Stage，再判断预算。`same_stage_repair_required` 表示当前 Stage 是最窄 owner：repair budget 仍可用时 reviewer 是非终局 Attempt，只可返回 `route_impact.stage_route_recommendation`，由 Controller 创建 repairer。`cross_stage_route_back_before_budget_exhaustion` 仅在另一个 declared Stage 是最窄 canonical owner 时成立：reviewer 可保留 `outcome=repair_required` 并提前返回 terminal `route_impact.stage_route_decision`，但必须使用 `decision_kind=route_back`、不同于当前 Stage 的 `target_stage_id` 和非空 `evidence_refs`。这是 `repair_required` 在预算耗尽前唯一合法的 terminal route；不得以 advance、skip、repeat、complete 或 route-back 当前 Stage 提前终局。若 repair budget 已耗尽且 artifact 可消费，该 reviewer 是 terminal decisive Attempt：保留 `outcome=repair_required`，返回 `route_impact.stage_route_decision`，并用未关闭 required findings 与 quality-debt refs 支撑路由；Controller 以 `completed_with_quality_debt` 终局，并将 StageRun 投影为 `completed_with_quality_debt`。`blocked`、`human_gate` 或零可消费 artifact 不返回 decision 或 recommendation，只返回对应 hard-boundary evidence；换言之，零可消费 artifact 不返回 route decision/recommendation。其他 terminal reviewer 返回 `route_impact.stage_route_decision`，必要时以证据 route-back 到最窄 declared owning Stage。Reviewer closeout 只返回 outcome、findings 与 receipt 所需审阅字段；正式 Review receipt 由 StageRunController 物化。Controller 将 `pass`、`repair_required`、`quality_debt` 分别映射为同名 receipt verdict，并将 `blocked` 或 `human_gate` 映射为 `hard_stop`；`hard_stop` 不是 Attempt outcome。

## Repairer

在新 thread 中只消费被审 artifact、accepted finding refs、source/rubric 与必要 lineage，在 owning Stage 边界内修复。逐 `finding_id` 返回含 `repair_status`、`changed_artifact_refs` 和 `repair_evidence_refs` 的 repair map，并返回 exact changed hashes 与 lineage。Repairer 不关闭 findings、不做终局 Stage 判断，只可返回 `route_impact.stage_route_recommendation`，不得返回 `stage_route_decision`，也不得写 target truth、artifact body authority、owner receipt 或 OPL runtime state。

## Re Reviewer

在另一个新 thread 中，消费 prior findings 与 repair map，用同一 source/rubric 对 exact changed artifact refs/hash 逐条关闭 accepted findings。质量结果只写入 `route_impact.stage_quality_cycle.outcome`，取值必须是 `pass`、`repair_required`、`quality_debt`、`blocked` 或 `human_gate`；同时返回整体 `evidence_refs`、remaining quality-debt refs，并为每个 accepted `finding_id` 输出 `closed`、`partially_closed` 或 `still_open`。不得返回独立 receipt `verdict`。只有 `required_finding_not_closed`、`repair_regression` 或 `critical_new_finding` 可以触发下一轮 Repair；触发 repair 的 regression / critical-new finding 必须 `required=true`，且 prior/open/new 集合中的 `finding_id` 必须唯一。普通新建议只能记录为 `optional_observation` 或明确的 quality debt，不得重开循环；`optional_observation` 本身不强制 outcome 降级，`pass + optional_observation` 合法。不得继承 repair rationale，也不得按 repairer 自述关闭 finding。先判断仍开放 required work 的最窄 canonical owner Stage，再判断预算。`same_stage_repair_required` 表示当前 Stage 是最窄 owner：`quality_round_index < max_repair_rounds` 时 re-reviewer 是非终局 Attempt，只可返回 `route_impact.stage_route_recommendation`，由 Controller 创建下一 repairer。`cross_stage_route_back_before_budget_exhaustion` 仅在另一个 declared Stage 是最窄 canonical owner 时成立：re-reviewer 可保留 `outcome=repair_required` 并提前返回 terminal `route_impact.stage_route_decision`，但必须使用 `decision_kind=route_back`、不同于当前 Stage 的 `target_stage_id` 和非空 `evidence_refs`。这是 `repair_required` 在预算耗尽前唯一合法的 terminal route；不得以 advance、skip、repeat、complete 或 route-back 当前 Stage 提前终局。若 `quality_round_index = max_repair_rounds`，即 repair budget 已耗尽且 artifact 可消费，该 re-reviewer 是 terminal decisive Attempt：保留 `outcome=repair_required`，返回 `route_impact.stage_route_decision`，并用仍开放 findings、repair regression 或 critical finding refs 支撑路由；Controller 以 `completed_with_quality_debt` 终局，并将 StageRun 投影为 `completed_with_quality_debt`。`blocked`、`human_gate` 或零可消费 artifact 不返回 decision 或 recommendation，只返回对应 hard-boundary evidence；换言之，零可消费 artifact 不返回 route decision/recommendation。其他 terminal re-reviewer 返回 `route_impact.stage_route_decision`，其 outcome 与 closure 字段由 StageRunController 物化为正式 Review receipt。
