# Stage architecture

Design a Stage graph in which each Stage owns one major open semantic judgment. Split distinct owners, knowledge sources, gates, artifact contracts, handoffs, or failure routes. Model deterministic generation and validation as Stage-internal obligations rather than additional semantic Stages; OPL remains their execution owner.

Use `oma-design-basis-architecture`. Define action I/O, artifacts, content refs, capability requirements, authority, and memory policy without materializing files.
