# OMA Design Basis Architecture

Normalize mode, exact target identity, objective, acceptance criteria, non-goals, constraints, and source refs. A create request has no target version; takeover and improve requests bind one exact target version. Distinguish source-derived facts, target-owner requirements, assumptions, and reusable design patterns. Admit only traceable design claims.

Assign one major open semantic judgment to each Stage. Split when owner, knowledge source, quality gate, artifact contract, failure route, or handoff differs. Model deterministic generation and validation as Stage-internal obligations, without executing or materializing them in OMA. Return semantic refs and policies, never physical paths or execution instructions.
