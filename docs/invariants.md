# Invariants

1. `engineer-agent` is the only public OMA action.
2. `design|diagnose` are internal provider operations only.
3. OMA never owns FoundryRun state, materialization, evaluation execution, evidence persistence, versions, activation, or rollback.
4. OMA protocol objects contain no physical execution coordinates or protected test bodies.
5. OMA cannot remove or weaken retained evaluation cases and can only raise risk.
6. Diagnosis binds the exact request, blueprint, and evidence digests.
7. Provider completion is not qualification or production adoption.
8. Generated Agents cannot modify their own code, evaluation, permissions, authority, or activation pointer.
9. Platform failures remain in OPL retry/failure handling and never become OMA-authored Agent changes.
10. A no-change proposal preserves the exact current blueprint, including generation, and has an empty semantic diff.
