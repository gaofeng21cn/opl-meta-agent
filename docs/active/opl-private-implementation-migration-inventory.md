## 当前 clean truth

`opl-meta-agent` 是 OPL-compatible Foundry Agent / builder agent，不是 OPL Framework 内置模块。OPL Framework 持有 standard scaffold、Agent Lab、generated interface bundle、runtime、queue、attempt ledger、provider receipt、observability、registry、App workbench、work-order execute/absorb/cleanup 和 promotion gate。

`opl-meta-agent` 只保留 agent-building semantic pack、refs-only action outputs、candidate package / developer work-order / mechanism proposal materialization 和 minimal authority functions。Purpose-first owner-delta gate 要求这些输出先回答 target-agent 语义增量是什么、下一 owner 欠什么 delta / receipt / gate / blocker；没有明确 owner route、next forced delta、verification/no-forbidden-write 或 target owner receipt/typed blocker 时，只能返回 typed blocker。

本文只维护当前私有实现分类、active caller、保留 authority、可上收 OPL 子域和退役门槛；不保存分支名、逐日拆分过程、执行流水、proof snapshots 或完整 per-file closeout。退役门槛的机器入口是 `runtime/authority_functions/meta-agent-authority-functions.json#script_morphology_policy/script_to_pack_retirement_gates` 与 `tests/source-purity.test.ts`；本文解释 gate 语义，不替代机器合同。已移出的过程叙事只通过 [过程历史索引](../history/process/README.md) 读取。

## OPL primitive dependencies

- Agent Lab suite runner and evolution/promotion gate.
- Generated interface bundle and standard scaffold generator.
- Pack compiler default domain-pack materializer for bootstrap / real-target smoke specs.
- Registry/discovery and App/workbench drilldown live consumption.
- Stage attempt ledger, queue, provider receipt and observability.
- Target-agent generic handoff vocabulary across MAS/MAG/RCA and new Foundry Agents.

## Forbidden claims

- `opl-meta-agent` work order does not sign target owner receipt.
- Suite pass, schema completeness, generated surface readiness or mechanism proposal cannot promote default agent.
- OMA cannot write target domain truth, memory body, artifact body, quality/export verdict or owner receipt authority.
- Registration/App projection readiness is not live App rendering evidence.
