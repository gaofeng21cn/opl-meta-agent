## 当前 clean truth

`opl-meta-agent` 是 OPL-compatible Foundry Agent / builder agent，不是 OPL Framework 内置模块。OPL Framework 持有 standard scaffold、Agent Lab、generated interface bundle、runtime、queue、attempt ledger、provider receipt、observability、registry、App workbench、work-order execute/absorb/cleanup 和 promotion gate。`opl-meta-agent` 只保留 agent-building semantic pack、refs-only action outputs、candidate package / developer work-order / mechanism proposal materialization 和 minimal authority functions。Purpose-first owner-delta gate 要求这些输出先回答 target-agent 语义增量是什么、下一 owner 欠什么 delta / receipt / gate / blocker；没有明确 owner route、next forced delta、verification/no-forbidden-write 或 target owner receipt/typed blocker 时，只能返回 typed blocker。

本文只维护当前私有实现分类、active caller、保留 authority、可上收 OPL 子域和退役门槛；不保存分支名、逐日拆分过程、执行流水或 proof snapshots。已移出的过程叙事见 [OMA private inventory process ledger](../history/process/2026-05-30-oma-private-inventory-process-ledger.md)。

## Bad-smell flags

- Large script-level policy: materializer/helper can grow into policy owner if stable vocabulary stays in scripts instead of `agent/`, `contracts/`, `runtime/authority_functions/` or OPL primitives.
- Agent Lab runner leakage: scripts may call OPL Agent Lab but cannot become Agent Lab implementation.
- Promotion shortcut risk: mechanism patch proposal and suite pass remain proposal-only / gated.
- Domain-specific command risk: OMA must not add MAS/MAG/RCA-specific command families; targets provide standard handoff.
- Same executor self-review risk: independent Codex reviewer requires separate invocation/context/task/receipt.
- Tests must not treat suite pass, generated surface proof or contract completeness as default promotion or target domain ready.
