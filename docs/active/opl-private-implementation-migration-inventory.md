## 当前 clean truth

`opl-meta-agent` 是 OPL-compatible Foundry Agent / builder agent，不是 OPL Framework 内置模块。OPL Framework 持有 standard scaffold、Agent Lab、generated interface bundle、runtime、queue、attempt ledger、provider receipt、observability、registry、App workbench、work-order execute/absorb/cleanup 和 promotion gate。

`opl-meta-agent` 只保留 agent-building semantic pack、refs-only action outputs、candidate package / developer work-order / mechanism proposal materialization 和 minimal authority functions。Purpose-first owner-delta gate 要求这些输出先回答 target-agent 语义增量是什么、下一 owner 欠什么 delta / receipt / gate / blocker；没有明确 owner route、next forced delta、verification/no-forbidden-write 或 target owner receipt/typed blocker 时，只能返回 typed blocker。Stage 语义路由由 decisive Attempt 给出，OPL StageRunController 只物化 transition；本仓不保留 current-pointer、transition 或第二路由控制面。

本文只维护当前私有实现分类、active caller、保留 authority、可上收 OPL 子域和退役门槛；不保存分支名、逐日拆分过程、执行流水、proof snapshots 或完整 per-file closeout。退役门槛的机器入口是 `runtime/authority_functions/meta-agent-authority-functions.json#script_morphology_policy/script_to_pack_retirement_gates` 与 source-purity tests；本文解释 gate 语义，不替代机器合同。已移出的过程叙事只通过 [过程历史索引](../history/process/README.md) 读取。

- Generated/default caller consumption：OPL `agents interfaces --repo-dir <this-repo> --json`。
- Production-consumption refs-only ledger：OPL `runtime oma-production-consumption list --json`。
- Active script caller surface：`package.json#scripts`、TypeScript imports, shell invocations and test refs outside `tests/source-purity.test.ts` self-guard strings.
- Split-pressure audit：fresh line-count scan over `scripts/`, `tests/`, `contracts/`, `runtime/` and `agent/`.
- Rational refactor scope audit：first match the requested write-scope paths against tracked files. If the requested Python source/test paths are absent and only analogous TypeScript / JSON surfaces exist, treat the lane as `no_safe_semantic_split` until the write scope is explicitly realigned; do not infer permission to edit adjacent TS/JSON files.

## Cleanup parity retention boundary

当前 retention 结论来自 `npm run source-structure:json` 的分类摘要，不是 `ready`、
`parity_complete` 或物理删除授权。strict line-budget 与 retention classification 是同一
readback 中的不同维度，不再维护独立 compact/full cleanup surfaces。

| Cleanup item | Current gate / readback route | Result | Owner route / required primitive | Tombstone stop condition |
| --- | --- | --- | --- | --- |
| `6` `scripts/execute-external-work-order.ts` | OPL owns work-order materialization/admission/execute/dry-run/absorb/cleanup/owner-closeout readback；OMA wrapper、package/action/schema/test caller均已删除。 | `retired` | `opl work-order materialize-request` + `opl work-order execute` | No-resurrection provenance recorded；source-purity receipt no longer scans the path. |
